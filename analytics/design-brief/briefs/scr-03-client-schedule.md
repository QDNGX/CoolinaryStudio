# SCR-03 — Расписание классов

| | |
|---|---|
| **Роль** | Клиент / Гость (просмотр доступен без входа) |
| **Платформа** | Веб (адаптив: десктоп + мобильный браузер) |
| **Приоритет** | Must (формально требование о витрине — Should, но без неё невозможен базовый сценарий брони) |
| **Реестр** | [screen-registry-cooking.md](../screen-registry-cooking.md) |

## 1. Кто и зачем сюда приходит

Человек, который хочет провести вечер за плитой в хорошей компании: он пришёл с главной страницы, по ссылке от друга или из закладки — посмотреть, «что готовят на этой неделе». Возможно, он ещё ни разу не был в студии и просто присматривается; возможно, это постоянный гость, который ищет свободный вечер под любимого шефа. Настроение — предвкушение и любопытство: он выбирает событие, а не оформляет заявку. Важно не спугнуть его требованием войти раньше времени — смотреть расписание можно свободно.

## 2. Работа экрана

Когда я думаю, как провести ближайшие дни, я хочу быстро увидеть, какие классы идут на этой неделе — что готовим, когда и с каким шефом, — чтобы выбрать класс и перейти к его карточке.

## 3. Входы и выходы

- **Приходит из:** главная страница / навигация сайта, прямая ссылка (в том числе от друзей и из соцсетей), возврат из карточки класса (SCR-04). Экран открыт и неавторизованному гостю — это витрина студии.
- **Уходит на:** карточка класса (SCR-04) — по выбору любого класса. Если гость без входа дойдёт до бронирования, вход (SCR-01) потребуется на шаге брони, а не здесь; расписание не должно ставить барьер входа перед просмотром. После входа клиент возвращается к прерванной броне, а не на общее расписание — гостевой доступ и такой возврат подтверждены (решение Р-16 п. 2, см. [журнал решений](../design-decisions-cooking.md)).

## 4. Что пользователь видит

| Приоритет | Элемент | Что показывает | Заметки для дизайнера |
|---|---|---|---|
| 1 | Лента классов на ближайшие 7 дней | Карточки предстоящих классов, сгруппированные по дням, от ближайшего | Это «афиша недели» — сердце экрана. День без классов можно не показывать вовсе или показывать свёрнуто |
| 2 | Карточка класса в ленте | Название программы, дата и время начала, шеф (имя + средний рейтинг), число свободных мест | Свободные места — живой сигнал: «осталось 2 места» подталкивает решать. Рейтинг шефа — компактно (например, звезда + число) |
| 3 | Фильтр по датам | Выбор периода дальше ближайших 7 дней | Вторичен: большинству хватает недели по умолчанию. Не должен визуально конкурировать с лентой |
| 4 | Отметка «класс отменён студией» | Отменённый класс либо скрыт, либо явно помечен как недоступный | Решение — за дизайнером (см. раздел 9); если показывается, запись с него должна быть очевидно невозможна |
| 5 | Служебное | Навигация сайта, вход/профиль | Для гостя — ненавязчивое приглашение войти, без давления |

## 5. Что пользователь делает

- **Основное действие (одно):** открыть карточку выбранного класса (переход на SCR-04).
- **Вторичные действия:** сменить период через фильтр по датам; вернуть период «ближайшие 7 дней»; перейти ко входу (для гостя — по желанию, не по требованию).
- **Что здесь сделать нельзя:** забронировать класс прямо из ленты — решение принимается в карточке, где есть вся информация (программа, шеф, прокат); записаться на класс, отменённый студией, — такие классы недоступны для выбора, и это должно читаться сразу, а не выясняться после клика.

## 6. Состояния экрана

| Состояние | Когда возникает | Что видит пользователь |
|---|---|---|
| По умолчанию | Есть классы на ближайшие 7 дней | Афиша недели: классы по дням, у каждого — программа, время, шеф с рейтингом, свободные места |
| Загрузка | Расписание запрашивается | Скелетон/плейсхолдеры карточек — сетка не «прыгает» при появлении данных |
| Пусто | На ближайшие 7 дней нет ни одного доступного класса | Empty state с текстом «Пока нет доступных классов» (формулировка фиксирована) и заметным предложением посмотреть более поздние даты через фильтр |
| Пусто (по фильтру) | В выбранном пользователем периоде классов нет | Тот же дружелюбный тон + быстрый способ сбросить фильтр к «ближайшим 7 дням» |
| Ошибка | Расписание не удалось загрузить | Человечное сообщение и действие «попробовать снова»; без технических деталей |
| Класс без мест | У класса не осталось свободных мест | Класс виден (интерес к студии не прячем), но состояние «мест нет» явно читается; карточку открыть можно |
| Класс отменён студией | Студия отменила класс | Класс недоступен для записи; скрыть или пометить — решает дизайнер (раздел 9) |

## 7. Правила, которые нельзя нарушать

- **ФТ-20 / US-15** — по умолчанию показываются классы на ближайшие 7 дней от текущего момента; более длинный период — только через фильтр по датам. Фильтр по датам — единственный фильтр витрины: фильтров по кухне, сложности или шефу в MVP нет (решение Р-11, см. [журнал решений](../design-decisions-cooking.md)). Empty state при отсутствии классов — с формулировкой «Пока нет доступных классов».
- **US-06 / ФТ-12** — рядом с шефом показывается его средний рейтинг (агрегат по отзывам): клиент выбирает класс осознанно уже на витрине.
- **D-04 / ФТ-05** — слот со статусом `CANCELLED_BY_STUDIO` недоступен для новой записи: из ленты не должно существовать пути к бронированию такого класса.
- **Правило доступа:** просмотр расписания доступен неавторизованному гостю; аутентификация (ФТ-13) требуется не для просмотра, а для создания брони (UC-01 выполняется только клиентом) — барьер входа встречает пользователя на шаге бронирования, не раньше; после входа клиент возвращается к прерванной броне. Подтверждено заказчиком (решение Р-16 п. 2, см. [журнал решений](../design-decisions-cooking.md)).
- **НФТ-01** — экран живёт в адаптивном вебе: афиша обязана хорошо работать на мобильном браузере, где её будут смотреть по дороге домой.

## 8. Тон и настроение

Это афиша гастрономических вечеров, а не таблица слотов. Хочется листать и предвкушать: «в четверг — паста с Марко, осталось три места». Тепло, аппетитно, по-человечески. Избегать канцелярита («доступные слоты», «записей не найдено») и ощущения кассы билетов — гость выбирает вечер, а не транзакцию.

## 9. Свобода дизайнера и открытые вопросы

- **Решает дизайнер:** компоновка ленты (список по дням / карточная сетка / календарная навигация); форма и место фильтра по датам; подача рейтинга шефа и свободных мест; показывать ли отменённые студией классы помеченными или скрывать их из витрины (обязательное условие — записаться на них нельзя); визуальные приёмы для «мест мало» / «мест нет»; микрокопирайт вне фиксированной формулировки empty state; момент и форма первого упоминания входа для гостя — лёгкая подсказка «для брони понадобится вход по email» заранее на витрине или первое упоминание только на шаге брони (правило зафиксировано: просмотр свободный, вход — на шаге брони, после входа — возврат к прерванной броне; решение Р-16 п. 2, см. [журнал решений](../design-decisions-cooking.md)).
