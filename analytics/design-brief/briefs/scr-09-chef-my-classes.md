# SCR-09 — Мои классы

| | |
|---|---|
| **Роль** | Шеф |
| **Платформа** | Веб (адаптив: десктоп + мобильный браузер) |
| **Приоритет** | Must |
| **Реестр** | [screen-registry-cooking.md](../screen-registry-cooking.md) |

## 1. Кто и зачем сюда приходит

Шеф — не «пользователь системы», а повар: он открывает этот экран с телефона на кухне, между
подготовкой продуктов и приходом гостей, часто мокрыми или занятыми руками. Аккаунт ему завёл
администратор, никакого онбординга у него не было — это первый и почти единственный экран,
который он видит после входа. Он хочет за пару секунд понять: какой у меня ближайший класс,
когда он начинается и сколько человек записалось, — и одним касанием провалиться в список
участников. Иногда он заходит из любопытства: посмотреть, как прошли прошлые классы и как его
оценивают гости.

## 2. Работа экрана

Когда я готовлюсь к своему классу или только что его провёл, я хочу мгновенно найти этот класс
в своём расписании, чтобы открыть список участников и работать с ним.

## 3. Входы и выходы

- **Приходит из:** экрана ввода кода (SCR-02) — это домашний экран роли «шеф» сразу после входа;
  возврат назад с экрана участников (SCR-10).
- **Уходит на:** SCR-10 «Участники класса» — по касанию на любой класс (и предстоящий, и
  прошедший). Другого «успеха» у экрана нет; выход из аккаунта — служебное действие.

## 4. Что пользователь видит

| Приоритет | Элемент | Что показывает | Заметки для дизайнера |
|---|---|---|---|
| 1 | Ближайший класс | Программа, дата и время начала, число записанных из общей вместимости (например, «7 из 12») | Самый крупный элемент экрана: если сегодня есть класс — шеф должен увидеть его, не скролля и не щурясь. Класс «сегодня» стоит явно пометить |
| 2 | Список предстоящих классов | Те же данные по каждому классу: программа, дата/время, записанные/вместимость | Сортировка от ближайшего к дальнему. Вся строка — одна большая цель для касания (руки заняты, точность низкая) |
| 3 | Счётчик записанных | «N из M» на каждом классе | Это главная рабочая цифра шефа: по ней он понимает объём закупки и подготовки. Полный класс можно акцентировать |
| 4 | Прошедшие классы | Тот же список, но завершённых классов; история показывается вся, без обрезания по давности (решение Р-16, см. [журнал решений](../design-decisions-cooking.md)) | Отдельная вкладка/секция, не смешивать с предстоящими. Нужны, чтобы отметить неявку уже после класса. Класс попадает в прошедшие сам, по времени — через 3 часа после начала (решение Р-10, см. журнал) |
| 5 | Пометка «отменён студией» | Если класс отменила студия — он остаётся виден с явной пометкой | Шеф не должен готовиться к отменённому классу; приглушить, но не прятать |
| 6 | Мой рейтинг | Средняя оценка шефа и число отзывов (например, «4,8 · 26 отзывов») — только агрегат, без текстов отзывов и оценок по отдельным классам (решение Р-07, см. [журнал решений](../design-decisions-cooking.md)) | Компактный блок в шапке/профиле, не конкурирующий с расписанием. Решение показывать его здесь зафиксировано в разделе 9. Блок не должен намекать, что его можно «развернуть» до отдельных отзывов, — их шеф не видит |
| 7 | Служебное | Имя шефа, выход из аккаунта | Минимально, не отвлекает |

## 5. Что пользователь делает

- **Основное действие (одно):** открыть класс — переход к списку участников (SCR-10).
- **Вторичные действия:** переключиться между предстоящими и прошедшими классами; обновить
  список; выйти из аккаунта.
- **Что здесь сделать нельзя** (и почему это важно показать/спрятать): создавать, переносить,
  редактировать или отменять классы — расписанием управляет только администратор, у шефа не
  должно возникнуть даже иллюзии таких кнопок; видеть классы других шефов — экран показывает
  только «мои»; отмечать неявку прямо из списка — это осознанное действие внутри конкретного
  класса (SCR-10), а не свайп по строке расписания.

## 6. Состояния экрана

| Состояние | Когда возникает | Что видит пользователь |
|---|---|---|
| По умолчанию | Есть предстоящие классы | Ближайший класс крупно, ниже — остальные предстоящие; прошедшие — в своей вкладке/секции. Класс переходит из предстоящих в прошедшие сам, через 3 часа после времени начала, — никто ничего не нажимает (решение Р-10, см. [журнал решений](../design-decisions-cooking.md)) |
| Загрузка | Данные ещё не пришли | Скелетон списка, без «прыжков» вёрстки; узнаваемый силуэт карточек классов |
| Пусто (предстоящие) | Администратор ещё не назначил шефу ни одного класса | Дружелюбная заглушка: «Пока нет назначенных классов» + пояснение, что расписание формирует студия. Без кнопок-обманок |
| Пусто (прошедшие) | Шеф новый, ещё ничего не провёл | Нейтральная заглушка вроде «Здесь появится история ваших классов» |
| Ошибка | Сеть/сервер недоступны | Понятное сообщение без технического жаргона + крупная кнопка «Повторить» |
| Класс сегодня | В списке есть класс с сегодняшней датой | Явный акцент «Сегодня» на карточке ближайшего класса |
| Класс отменён студией | Студия отменила слот, на который шеф был назначен | Карточка остаётся в списке с пометкой «Отменён студией», визуально приглушена, переход внутрь возможен (посмотреть, кто был записан), но без рабочих действий |
| Нет отзывов | По шефу ещё нет ни одного отзыва | Блок рейтинга не показывает «0,0» — либо скрыт, либо нейтральное «Отзывов пока нет» |

## 7. Правила, которые нельзя нарушать

- **ФТ-14 / D-08** — шеф не регистрируется сам: аккаунт заводит администратор. На этом экране не
  должно быть никаких намёков на редактирование профиля, смену email или онбординг — только
  готовое рабочее место.
- **ФТ-03 / UC-07** — расписание (программы, слоты, вместимость) формирует администратор. Экран
  шефа строго read-only по отношению к расписанию.
- **ФТ-19** — у класса есть жизненный цикл (запланирован / отменён студией / завершён); отменённый
  студией класс нельзя показывать как рабочий — пометка обязательна. Завершение происходит
  автоматически по времени — через 3 часа после начала класса; ни у шефа, ни у администратора
  кнопки «завершить» нет (решение Р-10, см. [журнал решений](../design-decisions-cooking.md)).
- **ФТ-12** — рейтинг шефа и число отзывов — вычисляемый агрегат; экран только отображает его,
  никакого управления отзывами здесь нет. Шеф видит **только агрегат**: тексты отзывов и оценки
  по конкретным классам ему не показываются — их читает только администратор студии
  (решение Р-07, см. [журнал решений](../design-decisions-cooking.md)).
- **UC-06 (предусловие)** — этот экран — единственный путь шефа к отметке неявки: если шеф не
  найдёт свой класс здесь, требование ФТ-10 не выполнимо. Поэтому находимость нужного класса —
  критерий качества экрана.
- **НФТ-01** — адаптивный веб: основной сценарий — мобильный браузер, десктоп вторичен.

## 8. Тон и настроение

Рабочий инструмент повара, а не «личный кабинет сотрудника»: тёплый, крупный, спокойный.
Экран должен читаться с расстояния вытянутой руки на залитом светом кухонном столе. Избегать
канцелярита («слот», «бронирования»), таблиц с мелкими колонками и любого ощущения отчётности —
шеф здесь хозяин своей смены, а не подотчётное лицо.

## 9. Свобода дизайнера и открытые вопросы

- **Решает дизайнер:** компоновка «ближайший класс + список» (карточки, лента, календарная
  группировка по дням); способ разделения предстоящих и прошедших (вкладки, секции, фильтр);
  визуальный язык пометок «Сегодня», «Отменён студией», «Класс заполнен»; место и форма блока
  рейтинга; показывать ли классы, отменённые студией, и среди прошедших после их даты — или
  только среди предстоящих до момента их даты; микрокопирайт заглушек и ошибок.
- **Зафиксированное решение (рейтинг):** средний рейтинг и число отзывов **показываем на этом
  экране** — у шефа всего два экрана, и другого «своего» места для агрегата ФТ-12 в интерфейсе
  роли нет; но это фоновая, а не рабочая информация: блок не должен конкурировать с расписанием
  и не должен выглядеть как KPI-отчёт перед руководством.
- **Открытые вопросы к команде:**
  - Открытых вопросов нет — все закрыты, см. [журнал решений](../design-decisions-cooking.md):
    шеф видит только агрегат рейтинга (Р-07), история прошедших классов показывается вся (Р-16),
    показ отменённых студией классов среди прошедших передан на усмотрение дизайнера.
