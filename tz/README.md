# ТЗ на приложение — «Шеф-стол»

Пакет технических заданий для разработки веб-приложения кулинарной студии «Шеф-стол».
Каждое ТЗ — самодостаточный документ для разработчика: сценарии, интерфейс и состояния
(из постановок на дизайн), контракт API (из [openapi.yaml](../api/openapi.yaml)),
бизнес-правила с ID и проверяемые критерии приёмки.

## Общие положения (действуют для всех ТЗ)

- **Платформа**: единое веб-приложение — адаптивный веб-фронт + бэкенд Spring Boot (БТ-01, НФТ-01); отдельного мобильного приложения нет.
- **Роли**: CLIENT / CHEF / ADMIN — три роли одного `User`; доступ к операциям — по матрице доступа ER-модели.
- **Аутентификация**: passwordless для всех ролей — email + одноразовый шестизначный код (БТ-06, ФТ-13); bearer-токен из `POST /auth/code/verify`; паролей нет ни у кого.
- **Гостевой доступ** (Р-16): расписание, карточка класса, каталог и контакты студии — без входа; вход требуется на шаге брони, после входа клиент возвращается к прерванной броне.
- **Контракт API** — [openapi.yaml](../api/openapi.yaml); семантика кодов: 409 — временный конфликт (можно повторить/выбрать другое), 410 — окончательная недоступность (не ретраить).
- **Производные поля** (`freeSpots`, `lateCancelCount`, `blockedUntil`, `averageRating`, `reviewsCount`, `isLateCancellation`, статусы `COMPLETED`) меняет только система по инвариантам D-01…D-11 — ни одна роль напрямую.
- **Лимиты текстов** (Р-16): причина отмены слота ≤ 300 символов, комментарий отзыва ≤ 1000, заметка об аллергии ≤ 500, фото программы ≤ 10.
- **Email-уведомления** — best-effort (D-07, НФТ-03): сбой отправки не блокирует и не откатывает транзакции.
- **Вне скоупа MVP** (БТ-08): онлайн-оплата, программа лояльности, push/мобильные уведомления, архивация программ и деактивация шефов (Р-15), редактирование слота (Р-03), редактирование отзыва (D-05), редактирование email (Р-16).

## Тон продукта

Тёплая гастрономия — «студия, куда хочется прийти», а не «система бронирования»
(из шаблона постановок). Тексты ошибок не обвиняют пользователя: «место только что заняли» —
честный отказ без вины (НФТ-02, sequence-диаграмма).
